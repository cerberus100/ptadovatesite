Our Commitment

True North Advocates Patient Advocates is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply relevant accessibility standards.

Accessibility Standards

WCAG 2.1 AA Compliance

Our website conforms to the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level. These guidelines explain how to make web content accessible to people with disabilities.

The Four Principles of Accessibility:

1. Perceivable

Information must be presentable in ways users can perceive, regardless of sensory abilities.

2. Operable

Interface components must be operable by all users, regardless of physical abilities.

3. Understandable

Information and operation of the interface must be understandable to all users.

4. Robust

Content must be robust enough to be interpreted by assistive technologies.

Accessibility Features

üîç Visual Accessibility

  • High Contrast: Sufficient color contrast ratios (4.5:1 for normal text, 3:1 for large text)
  • Scalable Text: Text can be enlarged up to 200% without horizontal scrolling
  • Clear Typography: Readable fonts with appropriate spacing and sizing
  • Focus Indicators: Visible focus indicators for keyboard navigation
  • Color Independence: Information not conveyed by color alone

Keyboard Accessibility

  • Keyboard Navigation: All interactive elements accessible via keyboard
  • Tab Order: Logical tab sequence throughout the site
  • Skip Links: Skip navigation links for screen reader users
  • No Keyboard Traps: Users never trapped in any interface element
  • Custom Controls: All custom interactive elements keyboard accessible

üîä Screen Reader Support

  • ARIA Labels: Descriptive labels for all interactive elements
  • Semantic HTML: Proper HTML structure for screen reader navigation
  • Alt Text: Alternative text for all meaningful images
  • Headings Structure: Logical heading hierarchy (H1-H6)
  • Form Labels: Clear labels and instructions for all form fields

üì± Mobile Accessibility

  • Touch Targets: Minimum 44px touch target size
  • Responsive Design: Accessible across all device sizes
  • Gesture Alternatives: Alternative methods for complex gestures
  • Orientation Support: Works in both portrait and landscape
  • Voice Control: Compatible with voice navigation software
